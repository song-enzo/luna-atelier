Overview

项目概述

LUNA ATELIER 是一套运行在极空间 NAS 上的完整女装定制业务系统，配合多Agent智能团队实现自动化运营管理。

🧵 核心系统

LUNA ATELIER 订单系统

基于 Python Flask + SQLAlchemy 的全功能服装定制订单管理平台。客人可在手机上浏览款式、选择面料花色、提交定制订单；管理员在后台进行订单处理、款式管理、用户管理。系统部署在 NAS Docker 容器中，响应式设计支持移动端操作。

🤖 智能团队

多Agent分工体系

由 Hermes Agent 驱动的多角色AI团队：小天（总调度/生活助理）、程序员（OpenCode+Kimi API 负责代码维护）、美工（通义万相负责图片生成）、策划师（负责方案规划）。Agent之间通过 Kanban 看板任务分配和 Telegram 协作。

🎯 系统定位： 用户是一位在意大利 Prato 的女装制版师（西装），技术能力强，追求效率。LUNA ATELIER 系统服务于其服装定制店面业务，多Agent团队则覆盖了从代码开发、设计出图到运营策划的完整技术支持链条。

User Flows

多角色操作流程

不同角色在系统中的完整操作路径。

👤 客人操作流程

1

获取账号

联系 LUNA ATELIER（管理员）创建客人账号，获得用户名和初始密码（默认 123456）

2

登录系统

通过浏览器访问 http://192.168.1.22:5000，使用账号密码登录

3

浏览款式

在款式列表页查看全部42款服装，可按品类筛选（西装/连衣裙/大衣/上衣/裤装/半裙/套装）或搜索款式名/编号

4

选择款式 & 下单

进入款式详情页 → 选择尺码 → 添加花色（预设色/上传面料图片/Canvas裁剪花色） → 填写数量 → 添加备注 → 提交订单

5

跟踪订单

在「我的订单」列表查看所有订单及当前状态（待确认 → 已确认 → 裁剪中 → 车缝中 → 质检中 → 运输中 → 已完成）

6

确认收货

当订单状态为「运输中」时，点击「确认收货」完成订单。系统自动记录操作日志

🔧 管理员操作流程

1

登录后台

使用管理员账号登录，自动跳转至管理仪表盘

2

查看仪表盘

概览总订单数、待处理订单、生产中订单、运输中订单、已完成订单，以及最近10条订单

3

订单处理

在订单列表筛选/搜索 → 进入订单详情 → 审核订单信息 → 设置金额和已付金额 → 进行状态流转操作 → 可选编辑花色明细

4

款式管理

添加新款（名称/编号/描述/面料信息/价格/图片） → 编辑现有款式 → 下架（软删除）款式

5

客人管理

创建新客人账号（用户名/昵称/手机号/密码） → 查看客人列表 → 管理客人状态

🤖 Agent 调度流程

1

用户发起需求

通过 Telegram 与小天对话，提出需求（如「帮我写个自动化脚本」「生成一张西装试穿效果图」「写一份开店报告」）

2

小天分析调度

小天理解需求类型，决定调用哪个 Agent：编程任务→程序员，设计任务→美工，方案任务→策划师

3

执行任务

使用 dispatch 命令通过 Kanban 看板分配任务，目标 Agent 接收后执行：
• 程序员: opencode run '任务' --model kimi-k2.6
• 美工: 调用通义万相 API 生成图片 + Python Pillow 处理
• 策划师: 撰写方案文档（Markdown/HTML/PPTX）

4

结果返回

Agent 完成后将结果返回给小天，再由小天通过 Telegram 反馈给用户。如需交付文件，保存在 /opt/data/home/小天专用/

Data Flow

关键数据流

一次从客人下单到确认收货的完整数据流转。

①

客人提交订单

POST /guest/order/{style_id} → 前端收集 items (color_name, swatch_data(base64), qty) + size + remark → Flask 解析 → 创建 Order 记录 + OrderItem 明细 + OrderLog 日志 → 写入 SQLite → 返回订单ID

②

花色图片存储

用户上传面料图片 → POST /upload/fabric → 保存至 uploads/fabrics/{uuid}.jpg → 返回图片URL
Canvas 裁剪花色 → base64 编码 → 服务端解码 → 保存至 uploads/swatches/{uuid}.png → OrderItem 关联图片路径

③

管理员审核 & 金额设置

GET /admin/order/{id} → 加载订单详情 + 所有日志 → 管理员在表单中设置 total_amount / paid_amount → POST 更新 → 写入 SQLite

④

状态流转

管理员选择新状态 + 备注 → POST 提交 → Order.status 更新 → OrderLog 记录 (status_from → status_to, note, operator) → 写入 SQLite → 页面刷新显示新状态

⑤

客人确认收货

POST /guest/order/{id}/confirm → 校验订单属于当前用户 + 状态为 shipped → Order.status = 'completed' → 记录日志 → 重定向到订单详情

Roadmap

未来规划与扩展方向

基于当前系统架构和业务需求，建议的迭代优化方向。

🅿️ 高优先级

数据库迁移：SQLite → PostgreSQL

解决并发写性能和数据可靠性问题，支持更多订单同时操作，为开店做准备

🅿️ 高优先级

多语言支持（中/意/英）

Prato 地区华人客户多，后续如需扩展意大利本地客户，需要中意双语界面

🅿️ 中优先级

款式图片的AI自动生成

接入美工 Agent（通义万相），自动生成新款式展示图，减少人工找图时间

🅿️ 中优先级

在线支付集成

接入 PayPal / Stripe 支付网关，客人可在线付款，自动标记已付金额

🅿️ 中优先级

消息推送通知

状态变更时通过 Telegram 自动通知客人（待确认→生产中→运输中→已完成）

🅿️ 中优先级

款式分类标签系统

支持多标签分类、面料标签、风格标签，提升搜索和推荐效果

🅿️ 低优先级

虚拟试衣间（3D预览）

结合美工 Agent 能力，让客人上传身材照片后看到服装上身效果

🅿️ 低优先级

数据分析与报表

销售趋势、热门款式、客人偏好分析，辅助选品和库存决策

🅿️ 低优先级

NAS 外网安全访问

配置反向代理（Nginx）+ HTTPS + 域名绑定，支持客人远程访问下单

🅿️ 低优先级

多店 / 多管理员支持

如果未来开分店，支持多管理员角色分工、独立统计
